# quality-b — coverage gate (cargo-llvm-cov)

Cross-cutting infra task (rule 1 waived for CI/scripts/docs). No crate logic changed.

## What landed
- `.github/workflows/quality.yml` — `coverage` job hardened: dropped `continue-on-error`,
  installs stable + `llvm-tools-preview`, runs
  `cargo llvm-cov nextest --all-features --lcov --output-path lcov.info --fail-under-regions 93`,
  and uploads `lcov.info` as an artifact.
- `docs/CODE-QUALITY.md` — new `## Coverage baseline (2026-06-16)` table with real per-crate
  and workspace region/line numbers.
- `scripts/coverage.sh` — one instrumented `cargo llvm-cov nextest` run, then emits both
  `target/llvm-cov/lcov.info` and an HTML report. shellcheck-clean; runs on macOS + Linux.

## Measured baseline
Workspace **region** coverage = **93.86%** (3845 regions, 236 missed). FLOOR = `floor(93.86)`
= **93**, no margin padding beyond rounding down. Per-crate: unison 97.42%,
hypercall-proto 83.94%, snapshot-store 94.49%, vtime 99.42%.

## Verification (local, this machine)
- `scripts/coverage.sh` produces `lcov.info` + `html/index.html`.
- Gate has teeth: `cargo llvm-cov report --fail-under-regions 93` → exit 0; `--fail-under-regions 94`
  → exit 1.
- `git diff` touches only `.github/`, `scripts/`, `docs/`.

## Decisions / notes for the integrator
- **Region** floor (not line/function) per the spec and `docs/CODE-QUALITY.md` rationale.
- Floor tracks the *workspace* total, not the weakest crate; `hypercall-proto`'s decode-heavy
  unreached error branches are the main drag — raising it is organic test work (non-goal here).
- `Cargo.lock` is generated by the build but was untracked and is a workspace-root file
  (rule 1) — left untouched/uncommitted.
- Ratchet the floor upward as coverage improves; never round to a clean number.
