# `harmony-linux`

The first pluggable guest-environment tier: pinned Linux kernels and images,
guest-resident agents, and the internal no-std wire SDK. The engine's bare-metal
acceptance workloads and their goldens live at `consonance/acceptance-suite`, not
here; that carve-out keeps product tests with consonance while keeping OS policy
out of the deterministic substrate.

## Entry points

```sh
make -C harmony-linux fetch             # fetch and hash-check pinned sources
make -C harmony-linux libvoidstar       # clean-room compatibility library
make -C harmony-linux test-libvoidstar  # ABI + device-transaction unit gate
make -C harmony-linux test-linux        # Linux reproducibility + boot gate
make -C harmony-linux test              # library gate, then Linux gate

make -C consonance/acceptance-suite test-payloads  # portable acceptance gate
```

Linux artifacts land in `harmony-linux/build/`. Native build trees use the fixed
path `/tmp/harmony-linux-build` (override with `GUEST_BUILD_ROOT`). Task 43 changed
the old hashed path and kernel build identity only because `/dev/harmony` already
requires a kernel rebaseline; `linux/MANIFEST.sha256` is therefore regenerated by
the twice-reproducible Linux gate rather than silently retaining old hashes.

## Linux prerequisites

On Debian/Ubuntu:

```sh
apt-get install -y build-essential flex bison libelf-dev libssl-dev bc cpio \
  kmod wget qemu-system-x86 xz-utils bzip2 shellcheck
```

On macOS, run the Linux gate in a Linux/amd64 container as documented in
`docs/BUILDING.md`; it fails fast instead of skipping. The compatibility-library
gate itself is portable and uses the host C compiler.

## Environment-owned components

- `linux/` pins and builds the kernel and all reproducible initramfs variants.
  Its built-in `/dev/harmony` driver serializes the fixed hypercall pages, stamps
  pid/comm/userspace-RIP attribution, emits JSON synchronously, and obtains seeded
  entropy with a one-byte-write/eight-byte-read transaction.
- `libvoidstar/` is a clean-room implementation of the SDK-facing dynamic ABI.
  It never falls back to host entropy; device failures drop an event or return zero.
- `sdk/` is internal binary-wire plumbing for bare-metal payloads and guest agents.
- `flow-agent/` and `play-agent/` are Linux workload agents and remain standalone
  Cargo workspaces.

Pins, image-specific prerequisites, boot arguments, and determinism rationale are
recorded in `linux/versions.lock` and `linux/IMPLEMENTATION.md`.
