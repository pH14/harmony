# Strategic review — 2026-07 (Fable)

A full review of everything since hypervizor PR #11: the entire VMM bring-up, the box frontier
(real-Linux → Postgres → runc → k3s determinism), the dissonance design + crates, and the
foreman process. Five parallel subsystem deep-dives ground this document; file:line citations
are as of `origin/main` @ `9d60c75` (task 49). This document is the source material for the
Wave-4 specs (tasks 58–62, 94) and the next ROADMAP rewrite (task 62).

## Headline

**The determinism thesis is proven at full stack height.** Bare payloads → real Linux boot →
bare Postgres → runc OCI → single-node k3s with an intra-guest CNI, each deterministic-twice
(bit-identical serial + `state_hash`), including SMP-built kernels (`maxcpus=1`). The
existential risk the project was founded on is retired.

**But the product loop has never run once.** Dissonance's four crates are ~85% built to high
quality — against an in-crate toy. Zero lines of `consonance/` depend on any `dissonance/`
crate. There is no control-transport server, no explorer→VM adapter, no fault enforcement on
any path, and no coverage producer. Every dissonance task (24/25/12/45/50/51) carved the
integration glue out as a "frontier non-goal," and no task ever picked it up. The glue was
structurally nobody's job. **Wave 4 exists to close that loop.**

## Verdicts on the tier abstractions

- **consonance as the deterministic substrate with a small verb API — right, validated.**
  The boundary survived three workload escalations without consonance learning what runs
  inside it. But the verb API exists only as wire types (`dissonance/control-proto`) and
  `Vmm` methods — nothing serves it, nothing external has ever called it (→ task 58). Two
  self-inflicted boundary violations: the ~1,240-line Linux bzImage loader inside the
  substrate (`consonance/vmm-core/src/vendor/x86/linux_loader.rs`, audit F1) and `vmm.rs`
  as a ~2,800 non-test-line god module with a five-bool implicit run-state machine
  (`vmm.rs:373`).
- **dissonance as the two-plane, two-loop explorer — the design is sound; it lacks contact,
  not design.** The two-plane litmus test, the Moment-keyed genesis-complete reproducer, and
  the "adding a fault never touches the outer loop" invariant match convergent field practice
  (FoundationDB seed-driven ∪ Antithesis coverage-guided as two backings of one artifact).
  Every open question (task 93's compose-vs-genesis-only, coverage geometry, StopMask
  semantics) is empirical and gets answered by the first real campaign, not by more prose.
- **`harmony-<env>` guest layers — task 43 supplies the first implementation.**
  `harmony-linux/` now owns the pinned kernel/images, agents, `/dev/harmony`, and the
  compatibility SDK library; acceptance payloads/goldens live on consonance's test surface.
  Workload-specific `/init` scripts remain a packaging debt, but the environment boundary is
  concrete and the kernel patch series is explicit and reproducibly gated.
- **x86 behind the VMM interface / ARM — not close; correctly deferred.** The `Backend`
  trait is a clean *substrate* seam (stock/patched/mock), not an *architecture* seam:
  `VcpuState`, `CpuidModel`, `MsrFilter`, and half the `Exit` variants are verbatim x86/KVM.
  `docs/ARM-PORT.md` is honest about this; the ARM gate remains the branch-count spike on
  real silicon, not a trait refactor. The one already-arch-neutral seam is
  `vtime::CpuBackend`.

## What holds up (keep doing this)

- The V-time engine is one coherent discrete-event clock: execute-to-event
  (`InjectionPlanner`, task 47) and jump-over-event (`IdlePlanner`/`resume_idle`, task 52)
  as duals over one `VClock`; integer-only, Kani-proven, adversarially simulated.
- Discipline is mechanized: zero TODO/FIXME in core src, no raw unwrap/panic outside tests,
  fail-closed default-deny on unmodeled exits/ports/MSRs, determinism clippy lints,
  strictly-additive hash-schema growth (old goldens never re-bless).
- `vm-state`'s TLV codec and unison's `state_hash` vs `observable_digest` split are exemplary.
- Known holes are disclosed inline and gate-caught (harmony#34 → patches 0004/0005, task 57).

## Ranked gaps

1. **The missing middle (systemic).** Delegation produced polished pure-logic leaves and
   heroic box bring-ups, with the integration layer structurally unowned. Fix: tasks 58–61.
2. **Snapshot substrate is O(image) per branch.** No KVM dirty-log harvester exists (live
   tests hand-supply dirty sets, `live_snapshot_branch.rs:210`); restore is a full guest-RAM
   memcpy (`vmm.rs:643`) plus a fresh tempfile per `materialize`
   (`snapshot-store/src/lib.rs:275`). Task 39 Gate 2 (dirty-proportional restore) is unmet by
   merged code. Fine for gates; a wall for campaign throughput. Deferred until task 60 makes
   it measurable (D5, below).
3. **The real Linux path traverses zero hypercall seams.** Entropy is RDRAND/RDSEED-exiting,
   storage is in-guest RAM ext4, console is UART — so `hypercall-proto`/`hypercall-doorbell`
   are unused by the flagship workloads, and the guest-plane model ("guest hits a service and
   parks") has **no in-band channel** on the path that matters. Consequences: (a) the first
   guest-plane vertical is the net path (task 61), which gives the hypercall stack its first
   real consumer; (b) block-fault injection is gated on D1 (host-side block model), not on
   wiring an existing seam — for Linux workloads, "block EIO first" is a mirage.
4. **The explorer↔environment `EnvCodec` seam does not bind.** Signature mismatch
   (`explorer::EnvCodec::compose(base, branch_local)` infallible 2-arg,
   `seam.rs:110` vs `environment::EnvCodec::compose(base, tail, at) -> Result`,
   `envcodec.rs:140`), and the real `compose` fails closed on exactly the common cases
   (Seeded bases, standing faults, `envcodec.rs:142-145`) — so the explorer's
   genesis-complete-reproducer gate has only ever passed against its toy codec. → resolve
   task 93 (genesis-only) **before** task 58 bakes the seam in.
5. **The single-vCPU axiom was silently cracked.** "One vCPU, period" is load-bearing in the
   CPU/MSR contract topology, DISSONANCE.md's "exactly one outstanding decision," and
   `Moment = InsnCount`. Task 56 shipped `CONFIG_SMP=y` (+`maxcpus=1`, ~1.7×/step cost) and
   no doc acknowledges it. Needs an explicit integrator ruling (single online vCPU is the v1
   contract; real multi-vCPU is out of scope until re-ruled) recorded by task 62. Note the
   flip side (from the 2026-07 fuzzer research): deterministic SMP is harmony's potential
   edge over Antithesis, which is single-core-pinned — the ruling should defer, not foreclose.
6. **Orphaned / vestigial code (~4–5k LoC):** `telemetry` (no producer wired; its own docs
   say the vmm-core seam is "documented, not built"), `hypercall-doorbell` (zero consumers —
   rescued by task 61), the `acceptance-suite` binary (drives only the toy registry), the typed
   `VcpuEvents` kept only to avoid moving a hash, and mutation-shaped code (`+`-vs-`|`
   chosen to appease cargo-mutants) that will trap refactorers.
7. **Doc/spec/process rot** (details + fixes in task 62): ROADMAP a full wave stale; docs/PLAN.md
   contradicts settled rulings while anchoring contract citations; `00-CONVENTIONS.md`
   describes a regime ~2/3 of the recent queue ignores (no frontier-task class);
   DETERMINISM-CORPUS still centers struck tasks 22/23; `IMPLEMENTATION.md` name collision
   (three unrelated tasks' diaries in the three most authoritative slots); HEAD skill commit
   dropped the pi reviewer in pr-review but left handoff/foreman mandating it; task 90 is
   ~95% executed but marked pending, leaving `hypervizor` strings in **hashed** build inputs
   (`harmony-linux/linux/lib-build.sh:48,59,60` — the task-43 landmine: documented-stale or
   rebaseline, never a casual rename); "task 91" is a phantom reference
   (`tasks/16:3`) — no such file exists and licensing is already done.
8. **One-box bus factor.** Single destructive contract baseline (`det-cfl-v1`) on one
   machine; task 92 (registry: probe → select → validate) is the fix, still deferred behind
   Wave 4.

## Wave 4 — "close the loop"

One organizing principle: **the smallest end-to-end bug-finding loop on real software**,
with everything not serving it explicitly deferred.

| # | Task | What it proves |
|---|---|---|
| 0 | reconcile local `main` ↔ `origin/main`; merge 56/57 impls | `main` runs its own frontier |
| 93 | resolve compose-vs-genesis-only (**now**, evidence favors genesis-only) | the reproducer model is real, not toy-tested |
| 58 | control server in vmm-core + socket-backed `Machine` adapter, seed-driven only | the explorer drives a real VM: snapshot → branch(seed′) → run → hash → replay |

| 94 | rename: Variation→**Modulation**, Theme→**Progression** (and code's Timeline/Multiverse → same) | one vocabulary across docs/specs/code |

Explicitly deferred (recorded so they aren't re-derived):

- **D5 — snapshot performance** (dirty-log harvest + memslot-remap restore): first perf wall
  after task 60 exists; do it when campaign throughput demands it, informed by measured
  branch rates.
- **SDK / coverage-guided epoch**: highest-leverage dissonance component long-term (see the
  2026-07 fuzzer research: cell archive over AFL-style novelty; log/event-template +
  sometimes-assertion feedback channels; bursty/regime-based entropy) — but strictly after
  the seed-driven loop works. The explorer's current AFL-shaped corpus should not receive
  further investment before that redesign.
- **ARM port** (until the branch-count spike on candidate silicon), **real multi-vCPU**
  (until re-ruled), **harmony-kubernetes** (until harmony-linux is a real crate, task 43),
  **D1 block model**, further workload bring-ups (k3s is credibility enough).

## Process note

The delegation machine served the substrate era well, and strained exactly where predicted:
the spec constitution doesn't cover frontier work, recent specs are hand-off notes between
box sessions rather than portable contracts, and one architectural decision (task 93) was
merged as a best-guess in the integrator's absence. Wave 4 is integration-shaped — more of
it is frontier/integrator work by nature. Task 62 amends the constitution; the foreman
should treat 58–61 as frontier-class from the start.
